# Руководство для AI-ассистента (Jules)

Привет! Этот файл — твой главный источник информации по проекту. Пожалуйста, следуй изложенным здесь правилам и используй его как список задач.

## 1. Контекст Проекта

- **Название:** Stealth Action Game (условное)
- **Суть:** 3D-аудио игра в жанре стелс-экшен.
- **Ключевая особенность:** **Игра предназначена для незрячих людей.** Это означает, что **звук — главный приоритет**. Все механики должны иметь четкую звуковую обратную связь. Визуальное представление вторично и используется только для отладки.

## 2. ⚠️ Актуальные Задачи и Известные Проблемы

Этот список — наш текущий план работы.

### Критические Баги и Регрессии (Высокий приоритет)
*   **[БАГ] Отсутствует звук смерти NPC:** Звук, который проигрывался при смерти врага, пропал.
*   **[БАГ] Враги спавнятся слишком близко к игроку:** Начальная расстановка врагов происходит в непосредственной близости от игрока.
*   **[БАГ] Не слышно шагов бегущих врагов:** Отсутствует звуковая обратная связь при преследовании.
*   **[БАГ] "Прилипание" врагов:** В ближнем бою враги "приклеиваются" к игроку, не давая возможности для маневра.
*   **[БАГ] Слишком высокая скорость врагов:** Скорость бега врагов слишком высока, что делает побег невозможным.

### Необходимые Улучшения (Средний приоритет)
*   **[УЛУЧШЕНИЕ] Нет индикатора атаки:** Отсутствует четкий звуковой сигнал, предупреждающий об атаке врага.
*   **[УЛУЧШЕНИЕ] Звук "безопасной зоны" (тени):** Текущий звук эмбиента в тени нужно доработать, чтобы он лучше передавал ощущение безопасности.

### Новые Фичи (Низкий приоритет)
*   **[ФИЧА] Система реверберации:** Реализовать систему эха и реверберации для разных помещений. **Использовать OpenAL Soft.**
*   **[ФИЧА] Система помещений:** Реализовать двери и другие интерактивные элементы окружения со звуковым сопровождением.

## 3. Процесс Разработки

### Стиль Кода
*   **Язык комментариев:** Русский. Комментируй каждый логический блок, функцию, сложное место.
*   **Переменные:** `snake_case` (например, `detection_level`).
*   **Функции:** `CamelCase` (например, `HandlePlayerAttack`).
*   **Фигурные скобки:** Всегда на новой строке.

### Управление памятью
*   Используй умные указатели (`std::unique_ptr`, `std::shared_ptr`) вместо сырых указателей для управления ресурсами.

### Логирование
*   Для отладки используй `logError("СООБЩЕНИЕ");`. Не используй `std::cout` напрямую.

## 4. Сборка и Тестирование

### Зависимости и Установка
- **Язык:** C++17
- **Основные библиотеки:**
  - `SFML 3.0.1` (статическая линковка)
  - `OpenAL Soft` (для будущей системы реверберации)
- **Инструменты для тестов:** `xvfb`, `xdotool`

- **Команда для установки всех зависимостей в Linux (Ubuntu/Debian):**
  ```bash
  sudo apt-get update
  sudo apt-get install -y git cmake build-essential libx11-dev libxrandr-dev libxcursor-dev libxi-dev libgl1-mesa-dev libudev-dev libfreetype6-dev libvorbis-dev libogg-dev libflac-dev libopenal-dev xvfb xdotool

- **Команда:** Выполняется из корневой директории проекта.

  ```

## 5. Техническая Информация: SFML 3.0.1

-   `sf::SoundBuffer::loadFromSamples`: Требует 5-й аргумент — карту каналов (для моно: `{sf::SoundChannel::Mono}`).
-   `sf::Event`: Цикл событий теперь `while (auto event = window.pollEvent())`. Доступ к типу: `event->getIf<sf::Event::EventType>()`.
-   `sf::FloatRect::intersects`: Заменен на `findIntersection`.
-   `sf::VideoMode(w, h)`: Заменен на `sf::VideoMode({w, h})`.
-   `sf::FloatRect(x, y, w, h)`: Заменен на `sf::FloatRect({x, y}, {w, h})`.
-   `sound.setLoop(true)`: Заменено на `sound.setLooping(true)`.
-   `sf::Listener::setDirection`: `sf::Listener::setDirection(x, y, z)` заменено на `sf::Listener::setDirection({x, y, z})`.
-   `sf::Keyboard::isKeyPressed`: Для физических клавиш используется `sf::Keyboard::Scan::Code`.

## 6. Решение проблем со сборкой

В процессе разработки были выявлены и решены следующие проблемы со сборкой проекта.

### 1. Ошибка линковки: `undefined reference to fmt::...`

-   **Проблема:** Сборка прерывается с ошибками линковщика, который не может найти функции из библиотеки `{fmt}` (например, `fmt::v11::vformat`).
-   **Причина:** Проект использует библиотеку `OpenAL-soft`, которая, в свою очередь, зависит от библиотеки форматирования `{fmt}`. Однако в `CMakeLists.txt` не было указано, что эту зависимость нужно найти и подключить.
-   **Решение:**
    1.  Убедиться, что в системе установлен пакет для разработки `{fmt}` (например, `libfmt-dev` в Debian/Ubuntu).
    2.  Добавить в `CMakeLists.txt` следующие строки:
        ```cmake
        # Найти пакет fmt
        find_package(fmt REQUIRED)

        # ...

        # Добавить fmt::fmt в список библиотек для линковки
        target_link_libraries(StealthActionGame PRIVATE ... fmt::fmt)
        ```

### 2. Ошибка компиляции: `'alGenEffects' was not declared in this scope`

-   **Проблема:** Сборка прерывается с ошибками компилятора, который не может найти функции `alGenEffects`, `alEffecti`, `alGenAuxiliaryEffectSlots` и другие, связанные с EFX (Effect Extension).
-   **Причина:** Эти функции являются частью **расширения** OpenAL и не экспортируются стандартно. Их необходимо загружать динамически во время выполнения программы. Прямой вызов этих функций приводит к ошибке, так как их объявления не видны компилятору в глобальной области видимости.
-   **Решение:**
    1.  В `.cpp` файле, где происходит работа со звуком (`SoundEngine.cpp`), определить указатели на все необходимые функции EFX.
        ```cpp
        // В глобальной области видимости или namespace
        LPALGENEFFECTS alGenEffects = nullptr;
        // ... и так далее для всех нужных функций
        ```
    2.  В коде инициализации OpenAL, после создания контекста, проверить наличие расширения и загрузить функции.
        ```cpp
        if (alcIsExtensionPresent(openalDevice, "ALC_EXT_EFX") == AL_FALSE) {
            // Обработать случай, когда EFX не поддерживается (например, отключить реверберацию)
            return;
        }

        // Загрузка указателей
        alGenEffects = (LPALGENEFFECTS)alGetProcAddress("alGenEffects");
        // ... и так далее для всех указателей
        ```
    3.  Далее в коде использовать эти указатели вместо прямых вызовов функций.
