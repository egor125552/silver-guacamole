## 2. Инструкции по сборке

Этот проект использует CMake, поэтому процесс сборки стандартизирован для разных платформ.

### Сборка в Windows (с MinGW G++)

Предполагается, что у вас установлен **CMake** и **MinGW** (с компилятором G++).

1.  **Создайте `build.bat`:** Создайте в корневой папке проекта файл `build.bat` со следующим содержимым:

    ```bat
    @echo off
    REM =================================================================
    REM Скрипт для сборки проекта с помощью CMake и MinGW
    REM =================================================================

    REM Убедитесь, что CMake и MinGW (G++) добавлены в системную переменную PATH.

    REM --- Настройки ---
    SET CMAKE_GENERATOR="MinGW Makefiles"

    REM --- Сборка ---
    echo [1/3] Создание папки для сборки...
    if not exist build mkdir build
    cd build

    echo [2/3] Конфигурация проекта с помощью CMake...
    cmake .. -G %CMAKE_GENERATOR%

    echo [3/3] Компиляция проекта...
    cmake --build . --config Release

    echo.
    echo =================================================================
    echo Сборка завершена!
    echo Исполняемый файл должен находиться в папке: build\StealthActionGame.exe
    echo =================================================================
    pause
    ```

2.  **Запустите `build.bat`**: Двойной клик по файлу запустит процесс сборки.

### Сборка в Linux (Ubuntu/Debian)

1.  **Установите зависимости и SFML 3.0.1:** Если SFML 3.0.1 не установлена, используйте следующий скрипт:

    ```bash
    # Установка зависимостей, включая утилиты для тестирования
    sudo apt-get update
    sudo apt-get install -y git cmake build-essential libx11-dev libxrandr-dev libxcursor-dev libxi-dev libgl1-mesa-dev libudev-dev libfreetype6-dev libvorbis-dev libogg-dev libflac-dev libopenal-dev xvfb xdotool

    # Клонирование, сборка и установка SFML 3.0.1
    git clone --branch 3.0.1 https://github.com/SFML/SFML.git /tmp/sfml_source
    cd /tmp/sfml_source
    mkdir build && cd build
    cmake .. -DSFML_STATIC_LIBRARIES=ON
    make -j$(nproc)
    sudo make install
    sudo ldconfig
    ```

2.  **Соберите проект:** Выполните следующие команды из корневой папки проекта:
    ```bash
    rm -rf build
    mkdir build
    cd build
    cmake ..
    make
    ```
    Исполняемый файл `StealthActionGame` появится в папке `build`.

## 3. Пути к библиотекам (vcpkg / system-wide)

-   **Windows:** Пользователь использует `vcpkg` для управления зависимостями. `CMakeLists.txt` должен быть настроен для автоматического поиска пакетов, установленных через `vcpkg`, с помощью `CMAKE_TOOLCHAIN_FILE`.
-   **Linux:** В данной среде зависимости (включая SFML) были установлены системно (в `/usr/local/lib`, `/usr/local/include`). `CMake` находит их автоматически.

В обоих случаях **не следует прописывать пути к библиотекам в коде или `CMakeLists.txt` вручную**.

## 4. Ключевые изменения API для SFML 3.0.1

В процессе работы код был обновлен до SFML 3.0.1. Вот ключевые исправления, которые были внесены:

-   `sf::SoundBuffer::loadFromSamples`: Функция теперь требует 5-й аргумент — карту каналов. Для моно-звука используется `{sf::SoundChannel::Mono}`.
-   `sf::Event`: Цикл событий теперь использует `while (auto event = window.pollEvent())`. Доступ к типу события осуществляется через `event->getIf<sf::Event::EventType>()`.
-   `sf::FloatRect::intersects`: Метод заменен на `findIntersection`. Проверка `rect1.intersects(rect2)` заменена на `rect1.findIntersection(rect2)`.
-   Конструктор `sf::VideoMode`: `sf::VideoMode(w, h)` заменен на `sf::VideoMode({w, h})`.
-   Конструктор `sf::FloatRect`: `sf::FloatRect(x, y, w, h)` заменен на `sf::FloatRect({x, y}, {w, h})`.
-   `sound.setLoop(true)`: Заменено на `sound.setLooping(true)`.
-   `sf::Listener::setDirection`: `sf::Listener::setDirection(x, y, z)` заменено на `sf::Listener::setDirection({x, y, z})`.
-   `sf::Keyboard::isKeyPressed`: Для проверки физических клавиш теперь используется `sf::Keyboard::Scan::Code`, а не `sf::Keyboard::Key`.
