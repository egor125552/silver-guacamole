# Полуночная сортировочная

Доступная браузерная пространственная аудиоигра о ночной смене в автоматизированном грузовом терминале. Нужно найти четыре перегревающихся энергоядра, доставить их к центральному лифту, пользоваться охлаждением, управлять железнодорожными переключателями, выбирать безопасные или короткие маршруты и отвлекать дронов ограниченным запасом металлических болтов.

## Технологии

- TypeScript 5.8.3 и Vite 8.1.5;
- Phaser 4.2.1: сцена, игровой цикл, Arcade Physics, карта, объекты и столкновения;
- Aaja Audio Engine, commit `4ad11f77b15163699d9213bc461d8aefb37c12f7`, как единственный production-аудиограф;
- Playwright 1.61.1;
- FFmpeg для воспроизводимой подготовки и проверки аудио;
- GitHub Actions и GitHub Pages.

В репозитории находятся обычные TypeScript-исходники, закреплённый commit Aaja, воспроизводимые скрипты аудиосборки и лицензии. Сгенерированные бинарники не маскируются в base64 и не хранятся в bootstrap-архивах: `npm run engine:materialize` создаёт локальный `vendor/aaja-package` и собирает точный Aaja/WASM из закреплённого commit, а `npm run audio:build` скачивает официальные Kenney CC0-архивы, проверяет их SHA-256 и создаёт 31 production OGG. CI и Pages выполняют эти шаги воспроизводимо и никогда не коммитят результаты обратно в ветки.

## Установка и проверка

Требуются Node.js 22+ и FFmpeg. Для независимой пересборки закреплённого Aaja нужны стабильный Rust, target `wasm32-unknown-unknown` и `wasm-pack 0.15.0`.

```bash
npm run engine:materialize
npm ci
npm run check
npm run test:e2e
npm run dev
```

`npm run engine:materialize` требует Git, стабильный Rust, target `wasm32-unknown-unknown` и `wasm-pack 0.15.0`. `npm run check` воспроизводимо собирает 31 аудиофайл из официальных архивов, а затем проверяет форматирование, lint, TypeScript, unit/contract-тесты, лицензии, хэши, структуру репозитория и production-сборку. `npm run build` отказывается работать без закреплённого реального WASM.

## Мир и игровой цикл

Карта имеет размер 32×24 клетки и шесть акустически различающихся зон: открытый двор, металлический ангар, технический коридор, охлаждающий отсек, лифтовая шахта и машинное помещение. В мире есть восемь дверей, разрушаемые барьеры, две схемы маршрута тележки, пять охлаждающих площадок, ремонтные станции, шкафчики с ресурсами и шесть дронов трёх типов.

Дроны используют A* и состояния `patrol`, `investigate`, `chase`, `search`, `return`, `stunned` и `dormant`. Они слышат шум, исследуют последнюю известную позицию, реагируют на болты и возвращаются к патрулю. После четвёртой доставки начинается 180-секундная блокировка с повторным включением силовых переключателей и выходом в диспетчерскую.

## Управление

### Клавиатура

По умолчанию: WASD или стрелки — движение и поворот, Shift — быстрый ход, Space/Enter — действие, Q — бросить болт, H — инструкция, I — статус, Escape — пауза, Shift+Escape — экстренный стоп звука. Привязки движения, действия и болта переназначаются и сохраняются.

### VoiceOver

Постоянная панель нативных HTML-кнопок предоставляет полный доступ к движению, поворотам, действию, болту, статусу, инструкции, паузе, restart и аварийному стопу. Удержание не требуется, порядок стабилен, фокус не переводится на canvas автоматически.

### Жесты без VoiceOver

Свайп вверх/вниз — движение, влево/вправо — поворот, касание — действие, двойное касание — болт, долгое касание — статус, три касания — пауза, четыре — инструкция. `pointercancel`, blur и потеря видимости очищают удерживаемые команды.
